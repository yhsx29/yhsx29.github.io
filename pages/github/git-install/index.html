安装

Git 官网：https://git-scm.com/

推荐文档：https://git-scm.com/book/zh/v2

第一步：双击下载后的Git-2.21.0-64-bit.exe，开始安装，点击[next]

第二步：选择安装路径，点击[next]

第三步：在窗口中选择组件，默认即可：

第四步：配置菜单目录，默认即可，点击[next]

第五步：选择默认编辑器，默认即可，点击[next]

第六步：[修改系统的环境变量]界面，默认第二个选项，点击[next]

  • 选项一，不会修改系统环境变量，但是Windows系统cmd命令行中无法使用git命令
  • 选项二，会将git相关程序加入系统环境变量中，使得Windows系统cmd命令行中可以使用git命令
  • 选项三，会将git相关程序以及一些其他的Unix工具加入系统环境变量，使得Windows系统cmd命令行中可以使用git以及Unix工具。要注意的是，这将覆盖Windows工具，如 “ find 和 sort ”。只有在了解其含义后才使用此选项。

第七步：默认选择SSH可执行文件，选择开源代码，点击[next]

第八步：配置行结束转换，默认选择第一项配置，点击[next]

  • 选项一，选项让Git能够自动转换文件中的换行符；签出到本地时转换为Windows下的换行符，提交到服务器时转换为Unix下的换行符
  • 选项二，选项让Git在签出到本地时不做转换，保留原始文件的换行符；提交到服务器时转换为Unix下的换行符
  • 选项三，让Git在签出到本地时和提交到服务器时都不做转换

第九步：配置终端模拟器以与 Git Bash 一起使用，默认选择第一项配置，点击[next]

  • 选项一，Git Bash将使用MinTTY作为终端模拟器，该模拟器具有可调整大小的窗口，非矩形选区和Unicode字体
  • 选项二，选项使用Windows系统cmd命令行来作为Git命令行

第十步：配置额外的选项，默认勾选前两个配置即可，点击[Install]

  • 选项一，启用文件系统缓存
  • 选项二，启用Git凭证管理器
  • 选项三，启用符号链接

第十一步：等待安装完成

​ 打开cmd 输入 git help 验证是否能够调用git说明，如可以调用，则安装成功

git bash 配置：

基础配置

打开 git-bash.exe ,配置用户名和邮箱

配置ssh

1.生成 rsa 密钥

$ ssh-keygen -t rsa -C "公钥名称" ，推荐使用注册邮箱

一直按回车

Enter passphrase (empty for no passphrase):

密码短语：为空直接回车即可

2.打开公钥文件复制全部内容，添加到git账号的ssh公钥上

git config 配置

配置 Git 的相关参数。

git clone 克隆

从远程仓库克隆一个版本库到本地。

# 默认在当前目录下创建和版本库名相同的文件夹并下载版本到该文件夹下

git init 初始化

初始化项目所在目录，初始化后会在当前目录下出现一个名为 .git 的目录。

# 初始化本地仓库，在当前目录下生成 .git 文件夹$ git init

git status 查看仓库状态

查看本地仓库的状态。

git remote 操作远程库

git branch 操作分支

操作 Git 的分支命令。

git checkout 检出

检出命令，用于创建、切换分支等。

git cherry-pick 合并

把已经提交的记录合并到当前分支。

git add 添加暂存

把要提交的文件的信息添加到暂存区中。当使用 git commit 时，将依据暂存区中的内容来进行文件的提交。

git commit 提交

将暂存区中的文件提交到本地仓库中。

注：文本编辑器如下： i 进入编辑页面 esc 退出编辑状态，退出后 ：wq 即（shift+;wq）退出选择页面，文件前面带#号的忽略

git fetch 更新

从远程仓库获取最新的版本到本地的 tmp 分支上。

git merge 合并

合并分支。

git diff 比较

比较版本之间的差异。

git pull 获取最新

从远程仓库获取最新版本并合并到本地。首先会执行 git fetch，然后执行 git merge，把获取的分支的 HEAD 合并到当前分支。

git push 推送

把本地仓库的提交推送到远程仓库。

git log 显示记录

显示提交的记录。

git reset 还原

git revert 撤销

生成一个新的提交来撤销某次提交，此次提交之前的所有提交都会被保留。

git tag 标签

操作标签的命令。

git mv 重命名

重命名文件或者文件夹。

git rm 删除

删除文件或者文件夹。
